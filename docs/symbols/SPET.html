SPET(source, session, params)
Syncapse Platform Event Track 1.1 Event track is a system for tracking events in Facebook Tabs and Applications. These actions are tracked using a client-side library and recorded within the Syncapse Platform. Clients with access to the Measure product within the Syncapse Platform can run reports on the tracked events.
Parameters:
{String} source
The URL of the App/Tab that is being tracked
{String} session Optional
The current session
{Object} params Optional
any parameters to be passed to the "page-view" event (also can include debug: true)
{Boolean} params.debug Optional, Default: "true"
If debug is true, no data will be sent to the Syncapse tracking server. Instead, all tracking event data will be output to the browser console using console.log.
{String} params.trackDomain Optional, Default: "track.platform.syncapse.com"
The tracking domain to which the Event Track class will send event data.
{String} params.trackPath Optional, Default: "/track.gif"
The path component for the tracking image.
{String} params.location Optional, Default: "document.location.href"
A 'location' data element to send to the server.
{String} params.fromVariableName Optional, Default: "from"
Event Track can track where a user has arrived from by using a named GET variable.
{Boolean} params.inFbml Optional, Default: "false"
If you are using Event Track in an FBML context (which is officially unsupported), you may want to set the parameter to 'true'.
{String} params.imageId Optional, Default: "spet_image"
If you are using Event Track in an FBML context (officially unsupported), Event Track needs a DOM image element with this ID.
{Boolean} params.skipOnLoadEvent Optional, Default: "false"
Event Track will fire a 'page-view' event automatically in the constructor. If this is not your desired behaviour, set this parameter to 'true'.
Returns:
{SPET} An instance of the Syncapse Platform Event Tracker.
